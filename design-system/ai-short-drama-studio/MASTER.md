# 剧创 AI 前端设计系统

> 页面级规范放在 `pages/[page-name].md`；页面规范只覆盖明确列出的条目，其余均继承本文件。

**版本：** Apple Fluid Workspace v2  
**基准日期：** 2026-07-16  
**产品定位：** 面向短剧创作者的可控生产工作台，而非通用后台管理系统。

## 1. 体验目标

- **安静而明确：** 首要操作始终最醒目，辅助信息后退，不用装饰抢注意力。
- **即时而连续：** 按下即反馈；状态更新不跳动；可逆操作沿相同空间路径返回。
- **媒体优先：** 真实镜头、候选版本和时间线是主角，界面材质只负责组织它们。
- **保持掌控：** 当前版本、候选版本、质量门禁、运行任务和未保存状态必须可见。
- **跨设备可用：** 桌面用于深度编辑；移动端保留查看、审核、任务处理和基础编辑能力。

## 2. 视觉系统

### 2.1 字体

使用平台系统字体，不加载远程字体：

```css
font-family: -apple-system, BlinkMacSystemFont, "SF Pro Display", "SF Pro Text",
  "PingFang SC", "Noto Sans CJK SC", "Microsoft YaHei", sans-serif;
```

| 层级 | 尺寸 | 字重 | 行高 | 字距 |
|---|---:|---:|---:|---:|
| 页面标题 | `clamp(1.75rem, 2.4vw, 2.5rem)` | 700 | 1.08 | `-0.035em` |
| 区块标题 | `1.125rem` | 650 | 1.25 | `-0.018em` |
| 卡片标题 | `0.9375rem` | 650 | 1.35 | `-0.01em` |
| 正文 | `0.875rem` | 400 | 1.58 | `0` |
| 辅助文字 | `0.75rem` | 500 | 1.45 | `0.01em` |
| 眉题 | `0.6875rem` | 700 | 1.2 | `0.08em` |

中文正文不得长期低于 12px；表单在移动端至少 16px，避免系统自动缩放。

### 2.2 色彩

| 角色 | 标准工作台 | 暗房模式 |
|---|---|---|
| 画布 | `#F5F5F7` | `#090B10` |
| 实体表面 | `#FFFFFF` | `#171A21` |
| 浮动材质 | `rgba(255,255,255,.78)` | `rgba(24,27,35,.78)` |
| 主文字 | `#1D1D1F` | `#F5F5F7` |
| 次文字 | `#515154` | `#B7BBC5` |
| 弱文字 | `#6E6E73` | `#8B919D` |
| 主操作 | `#0A66D9` | `#2997FF` |
| 成功 | `#14804A` | `#32D583` |
| 警告 | `#A15C00` | `#FDB022` |
| 危险 | `#D92D20` | `#FF6961` |

颜色不可作为唯一状态信号；状态必须同时包含文字或图标。

### 2.3 间距、圆角与深度

- 间距节奏：`4 / 8 / 12 / 16 / 20 / 24 / 32 / 40 / 48 / 64`。
- 控件圆角：`10–12px`；内容表面：`16px`；重点媒体与大面板：`20–24px`。
- 阴影只有三层：静止表面、浮动控件、模态层；同层组件不得随机改变阴影。
- 半透明只用于导航、工具栏、弹出层和模态层；内容卡片默认使用实体表面，避免玻璃叠玻璃。

## 3. 动效语言

### 3.1 默认反馈

- 指针按下立即缩放到 `0.98`，释放后从当前呈现值回到 `1`。
- 普通状态切换使用无过冲的阻尼感曲线：`cubic-bezier(.2,.8,.2,1)`，响应约 `180–320ms`。
- 只有来源于拖拽、甩动或真实动量的交互允许轻微回弹；普通菜单、卡片和提示不回弹。
- 进入与退出沿同一路径；弹出层以触发点为 `transform-origin`。
- 动画只使用 `transform` 与 `opacity`；加载指示器是唯一允许持续循环的界面动画。

### 3.2 无障碍降级

- `prefers-reduced-motion: reduce`：位移动画改为短交叉淡化，取消过冲和持续装饰动画。
- `prefers-reduced-transparency: reduce`：浮动材质变为近实体背景并取消模糊。
- `prefers-contrast: more`：提高表面不透明度、边框和焦点环对比度。

## 4. 组件规则

### 按钮

- 桌面最小高度 38px，触控环境最小 44px。
- 主按钮每个操作区最多一个；次按钮使用实体浅表面；幽灵按钮仅用于低风险辅助操作。
- 悬停不改变布局；按下反馈发生在 `pointer-down` 对应的 `:active` 状态。
- 破坏性操作使用红色，并保留明确确认和可追溯说明。

### 输入与选择器

- 标签始终可见，不能只依赖占位文字。
- 聚焦时使用主色边框和外扩焦点环；错误信息紧邻字段。
- 输入高度至少 42px；文本区域允许用户调整高度。

### 表面与列表

- 同类信息优先组成连续列表，用间距和分隔关系表达层级；避免每一行都变成独立卡片。
- 可点击整行必须有悬停、聚焦和按下反馈，并显示手型光标。
- 媒体表面使用真实项目素材；无素材时使用诚实、克制的占位状态，不伪造内容。

### 弹出层与模态层

- 桌面弹出层从触发器方向展开；移动端模态转为底部面板。
- 模态任务使用遮罩并降低背景层级；非阻断面板不加遮罩。
- 原生 `dialog` 负责焦点约束与 Escape 行为。

## 5. 布局规则

- 桌面：固定侧栏 + 浮动顶部工具栏；内容最大宽度 1480px。
- 平板与手机：四项主导航变为底部浮动栏；顶部保留品牌、路径与实时状态。
- 页面必须在 `375 / 768 / 1024 / 1440px` 验证，无非意图横向滚动。
- 镜头工作台在窄屏按“场景导航 → 画面 → 参数 → 任务状态”自然纵向排列，不再阻断访问。
- 固定或浮动控件必须为内容留出安全区，并考虑 `env(safe-area-inset-*)`。

## 6. 文案与图标

- 可见文案默认使用自然简体中文；仅模型名、编码和必要技术名词保留英文。
- 标签具体描述内容或动作，例如“生成任务”“比较版本”，避免“首页”“更多”等模糊词。
- 全站只使用 Lucide 图标；不使用 Emoji 充当功能图标。
- 图标必须与文字或无障碍名称配对，装饰图标标记为 `aria-hidden`。

## 7. 交付门槛

- 键盘可完成主流程，并提供“跳到主要内容”链接。
- 焦点环清晰，无键盘陷阱；正文和控件达到 WCAG AA 对比度。
- 触控目标至少 44px；移动端无整页拦截、无隐藏在固定导航后的内容。
- 三种界面模式均通过浏览器检查；控制台无错误、资源 404 或 React 警告。
- 运行 `npm run lint`、`npm test -- --run`、`npm run build`，再在真实浏览器完成项目页、任务页、分集页和镜头工作台冒烟测试。
